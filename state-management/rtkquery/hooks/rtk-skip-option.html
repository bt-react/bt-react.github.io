    Skip Option

    In RTK Query, the skip option is typically used to conditionally prevent the execution of a query based on certain criteria. However, RTK Query allows only a single skip condition per query. If you want to use multiple conditions for skipping the query, you will need to combine the conditions in a logical way.

    For example, you can combine two conditions using logical operators (&& or ||) inside the skip field.

    Here’s how you can add two skip values to your query:

    const {
  data: approvalPersonData,
  isLoading: approvalPersonLoading,
  error: approvalPersonError,
} = useGetForexApprovalPersonQuery(
  {
    sellingAmount: formValues?.sellingAmount,
    sellingCurrencyId: formValues?.sellingCurrency,
  },
  {
    skip: !formValues?.sellingAmount || !formValues?.sellingCurrency, // Skip if either condition is not met
  }
);

    In the example above:

    • The query will be skipped if either formValues?.sellingAmount or formValues?.sellingCurrency is falsy.

    • You can adjust the logical condition (&&, ||) depending on your use case:

      • &&: Skip only if both conditions are falsy.
      • ||: Skip if either of the conditions is falsy.

    This allows you to apply multiple skip conditions in a single place, making your query conditional based on multiple factors.
